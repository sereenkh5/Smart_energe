Tools and Software Used

ThingsBoard

An integrated Internet of Things (IoT) platform used to collect, display, and analyze energy consumption data from multiple devices and sensors in real time. This platform is ideal for smart energy monitoring as it provides advanced tools to understand and improve energy usage.

Docker

A tool for packaging applications into containers that ensure independent and smooth operation on any environment. Docker is used in the system to connect cloud computing and various services (such as databases, ThingsBoard platform, servers) in an organized and scalable way.

Arduino IDE

An open-source development environment used to write and upload code to microcontrollers like the ESP32. It provides an easy interface to program sensors, read data, and convert signals into digital values.

Visual Studio Code

An advanced code editor that supports multiple programming languages and offers extensions to help write more complex code, such as Python, as well as tools for integrating Docker and managing software projects.

  • Code is written using Arduino IDE or VS Code (with PlatformIO or Arduino plugins) to upload firmware to the ESP32 microcontroller.
  • The code reads values from sensors (current, voltage, power) via ADC ports or suitable protocols.
  • Analog signals are converted into digital values, then formatted into suitable data structures.
  • An MQTT library is used in the code to connect to the internet and continuously send data to the MQTT broker via Wi-Fi.

  • MQTT is a lightweight messaging protocol suitable for resource-constrained devices.
  • After reading the data, the ESP32 publishes this data to an MQTT broker (a middleware server) under a specific topic.
  • Server applications like ThingsBoard subscribe to these topics and receive the data.
  • Other software or applications can also subscribe to the same topics to receive real-time data for analysis or user display.

  • Applications and services such as the MQTT broker, ThingsBoard platform, and databases are packaged inside Docker containers.
  • This allows these services to run independently and separately from the underlying system environment, enabling easy deployment and distribution.
  • Docker simplifies managing updates, scaling, and backups of these services.
  • Within the system, Docker connects the data sent from ESP32 via the MQTT broker and ensures its delivery to ThingsBoard and then to the graphical user interface (dashboard) and applications.

Model deployed on cloud platforms (Google Cloud or AWS) for scalability and real-time access.

Real-time predictions generated every 5 minutes for timely insights.

  • Training data from real ESP32 sensors
  • Preprocessing: resampling, normalization, outlier filtering
  • Tools: scikit-learn, Jupyter notebooks, pandas

Accuracy evaluated using Mean Absolute Error (MAE) and Root Mean Squared Error (RMSE).

Prediction accuracy decreases during sudden, random consumption events not following historical patterns.
