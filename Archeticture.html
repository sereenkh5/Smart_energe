System Architecture Diagram

Energy data is collected from sensors connected to the ESP32 (Esenoi 2 5730) via Wi-Fi and programmed using the Arduino IDE.

The data is transmitted using the MQTT protocol to an MQTT Broker within a Docker container, where it is processed using Python code developed in Visual Studio Code.

After initial processing, the data is stored in Cloud Storage and later utilized by an AI Predictive Engine to forecast energy consumption and analyze user behavior.

The results are presented through two main interfaces:

  • A ThingsBoard dashboard for companies to monitor consumption and generate analytical reports.
  • A dedicated mobile application for users, featuring a simplified and accessible interface for the elderly and people with disabilities. It also includes real-time alerts for abnormal consumption.

Communication between system components is established via Wi-Fi for wireless connectivity, MQTT for lightweight messaging, and REST API for integration with external applications and dashboards.

The system combines local processing (via Docker and Python) with cloud-based processing (Cloud Storage + AI) to deliver a smart, scalable solution with accurate reports and instant alerts that help improve energy management and reduce waste.

    Step-by-step operation of the system

    Workflow

  • Initial Input: The system starts with the ESP32 microcontroller reading sensor data from connected energy sensors (current, voltage, power). This reading happens automatically every second.
  • Data Acquisition: The sensors capture real-time electrical parameters. The ESP32 converts the analog signals from sensors into digital values using its ADC (Analog-to-Digital Converter).
  • Data Transfer: The ESP32 packages the sensor data into JSON format and publishes it using the MQTT protocol over Wi-Fi to an MQTT broker hosted inside a Docker container.
  • Data Processing: A backend Python script subscribes to the MQTT topics, receives the data, parses the JSON payload, cleans it, and performs necessary preprocessing (e.g., filtering out noise or outliers).
  • Algorithmic Decision-Making: The cleaned data is fed into an AI regression model deployed in the cloud, which predicts future energy consumption and detects any abnormal spikes or anomalies.
  • Data Storage & Visualization: Processed data and AI outputs are stored securely in cloud storage. The data is then sent via a REST API to a dashboard on ThingsBoard for real-time visualization by admins or companies, and also to a mobile application for end-users.
  • Final Outputs:
    • The dashboard updates graphs and statistics dynamically.
    • The mobile app notifies users of anomalies or energy usage alerts through push notifications.
    • Optionally, actuators or smart devices can be triggered based on AI predictions (e.g., to reduce power consumption).
  • Automation & Frequency: The entire process is automated — sensor readings and data transmissions occur every second, while AI predictions and dashboard updates happen every 5 minutes to balance responsiveness and resource efficiency.
