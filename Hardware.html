Hardware Layer

Physical components used to collect data from the environment

ESP32 Microcontroller

Wi-Fi enabled microcontroller that collects sensor data and transmits it wirelessly. Programmed using Arduino IDE

SCT-013 Current Sensor

Non-invasive current sensor that clamps around power cables to measure current consumption without direct electrical contact

ZMPT101B Voltage Sensor

Measures AC voltage levels with high accuracy and provides isolated measurement for safety

Raspberry Pi 4

Serves as the Docker host device running our MQTT broker and data processing services

Hardware Components & Process Flow

  • Solar Panel
    The main power source of the system.
    Converts sunlight into DC electricity to power components and charge the battery.
  • Diode
    Prevents current from flowing back from the battery or circuits to the solar panel at night or in low light.
  • DC-DC Converter
    Regulates the voltage coming from the solar panel to be suitable for the ESP32 and other components.
  • ESP32 Microcontroller
    The brain of the system.
    Receives data from sensors, processes it, and sends it via the internet to a server or platform (such as ThingsBoard).
    Supports Wi-Fi, making it ideal for IoT projects.
  • Energy Sensors (e.g., INA219)
    Measure voltage and current precisely.
    Allow calculation of power consumption or generation from the solar system.
  • Relay and Pump
    Relay: An electrically controlled switch operated by the ESP32 to turn devices on/off.
    Pump: An example of a load that can be controlled by the system.

  • Data Collection from Sensors
    The sensors (INA219) measure voltage and current.
    They send readings to the ESP32 via the I2C protocol.
  • Data Processing in ESP32
    The ESP32 calculates power = voltage Ã— current.
    It can also calculate total energy consumed or generated over time.
  • Sending Data to the Cloud
    The ESP32 transmits data via Wi-Fi using protocols like MQTT or HTTP.
    The data is sent to an IoT platform (such as ThingsBoard, AWS IoT, or Azure IoT).
  • AI Data Analysis
    The stored data in the cloud is processed by an AI model.
    The AI can:
    - Predict future energy consumption.
    - Detect anomalies or abnormal usage.
    - Optimize energy usage based on historical patterns.
  • Dashboard Visualization
    The IoT platform displays both real-time and historical data on an interactive dashboard.
    The user can view:
    - Live voltage and current readings.
    - Energy consumed/generated.
    - AI predictions for future usage.

  • AI algorithms analyze data to predict consumption patterns and detect anomalies.
  • Insights are displayed on an interactive dashboard for user decision-making and optimization.
